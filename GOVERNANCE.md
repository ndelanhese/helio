# Governance

Helio currently uses a maintainer-led model.

Maintainers set roadmap, review contributions, manage releases, and enforce project policies. Decisions should favor local-first behavior, operational safety, data ownership, portability, and maintainable scope.

Substantial changes start with an issue or discussion and require an approved design before implementation. Maintainers document meaningful architectural decisions in repository specs or decision records.

As contributor activity grows, maintainership may be granted based on sustained, constructive participation, sound judgment, and respect for safety and community standards.
